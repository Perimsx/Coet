# Coet 项目 Figma 设计稿说明

## 1. 项目定位

Coet 是一个“个人技术博客 + 后台运营系统”一体化项目。

它不是电商后台，也不是通用 CMS，而是围绕“个人内容输出、站点展示、评论互动、建议反馈、友链管理、站点设置”展开的轻量内容产品。

当前项目的核心特征如下：

- 前台负责个人品牌展示、文章阅读、内容归档、标签分类、友链展示、建议反馈。
- 后台负责文章文件管理、评论审核、建议工单、友链录入与巡检、关于页编辑、站点配置。
- 文章与关于页正文来自 Markdown/MDX 文件，不是数据库富文本 CMS。
- 评论、建议、友链、管理员会话、站点设置等运营数据存储在 SQLite。
- 产品气质应偏“克制、理性、清爽、可信、技术感”，而不是花哨营销风。

建议设计目标：

- 前台突出“内容可读性 + 个人品牌感”。
- 后台突出“运营效率 + 状态清晰 + 低学习成本”。
- 同一套品牌语言贯穿前后台，但前台更偏编辑感，后台更偏工作台。

## 2. 用户角色

### 2.1 前台用户

- 普通访客：浏览首页、文章、标签、分类、归档、关于、友链。
- 互动访客：提交评论、提交建议、申请友链。

### 2.2 后台用户

- 站长 / 管理员：登录后台、写文章、审核评论、回复建议、维护友链、编辑关于页、修改站点设置。

## 4. 视觉方向

### 4.1 色彩建议

基于当前项目样式，建议采用“浅色优先、蓝青系点缀、深色模式兼容”的方案。

主色建议：

- Primary Blue: `#2563EB`
- Sky Blue: `#0EA5E9`
- Cyan Accent: `#06B6D4`
- Success: `#10B981`
- Warning: `#F59E0B`
- Danger: `#EF4444`

中性色建议：

- Page Background: `#F8FAFC`
- Surface: `#FFFFFF`
- Soft Surface: `#F1F5F9`
- Border: `#E2E8F0`
- Text Primary: `#0F172A`
- Text Secondary: `#64748B`
- Text Tertiary: `#94A3B8`

深色模式建议：

- Dark Background: `#020617`
- Dark Surface: `#0F172A`
- Dark Surface Alt: `#111827`
- Dark Border: `#1E293B`
- Dark Text: `#F8FAFC`

### 4.2 字体建议

项目当前样式已经偏向以下组合，Figma 建议直接按此方向搭建：

- 中文 / 正文：`Noto Sans SC`
- 英文 / 标题：`Outfit`
- 代码 / 数据：`JetBrains Mono`

字号建议：

- H1: 48-72
- H2: 28-40
- H3: 20-28
- Body: 16-18
- Meta / Caption: 12-14

### 4.3 圆角与阴影

- 小圆角：12
- 中圆角：16
- 卡片常用圆角：24
- 大容器 / 大弹窗：28-32
- 按钮胶囊：999

阴影应偏柔和，不要重投影，不做厚重拟物。

### 4.4 背景与质感

- 前台整体可使用极浅网格点阵背景。
- 大区域可以加入淡蓝到白色的径向渐变。
- 顶栏和弹层可使用轻微毛玻璃与半透明边框。
- 后台页面适合用浅色渐变背景承托白色卡片，避免“纯白表格堆叠”。

### 4.5 动效建议

- 入场：轻微上移 + 渐显。
- 悬停：轻微抬升、边框提亮、阴影增强。
- 切换：标签页、筛选项、弹窗、抽屉保持 180-320ms。
- 大面积动画要克制，重点用于 Hero、卡片 hover、模态反馈、图表刷新。

## 8. 前台页面详细说明

### 8.1 首页 `/`

定位：

- 这是整个站点的品牌首页，要第一眼体现“这是一个有个人温度的技术博客”。

布局建议：

- 顶部为粘性导航。
- 首屏为全屏 Hero。
- Hero 桌面端采用左右双栏：左侧文案，右侧头像。
- Hero 下方为“最新文章 + 分类 + 热门标签”组合区。

模块构成：

- Logo
- 导航：首页、归档、项目、关于、友链
- 工具按钮：搜索、建议反馈、主题切换、移动端菜单
- 主标题：你好，我是 XXX
- 副标题：角色说明，例如“全栈开发者”
- 一句短标语
- 社交图标组
- 大头像
- 向下滚动提示
- 最新文章列表
- 分类列表
- 热门标签列表

视觉重点：

- Hero 要通透、简洁、有呼吸感。
- 头像区域要有轻微氛围光晕。
- 社交图标可做彩色圆形按钮，但不要过于花。
- 最新文章区要更偏编辑排版，不要做成电商卡片墙。

### 8.2 博客列表 `/blog`

定位：

- 全站文章索引页，要强调“内容沉淀感”和“分类检索效率”。

模块构成：

- 页面标题 + 文章总数
- 排序切换按钮
- 分类横向 Rail
- 文章列表
- 分页器

文章列表项信息：

- 发布时间
- 文章标题
- 文章摘要
- 主分类标签
- 标签组

设计要求：

- 分类切换建议做成横向可滚动胶囊。
- 列表项重排版、轻边界、强信息层级。
- 保持可读性优先，不要做过重封面卡片。

### 8.3 分类页 `/blog/category/[category]`

与博客列表共用同一套骨架，但要有明确“当前分类高亮状态”。

需要体现：

- 当前选中分类
- 当前分类文章数
- 仍可切换至其他分类

### 8.4 标签总览 `/tags`

定位：

- 标签索引页，应更轻、更灵活，像一个“主题地图”。

模块构成：

- 页面标题
- 标签总数与引用次数
- 排序切换
- 标签云 / 标签卡片墙

设计要求：

- 标签以胶囊或小卡片形式铺开。
- 每个标签带数量角标。
- Hover 可轻微上浮。

### 8.5 标签详情 `/tags/[tag]`

定位：

- 某个主题下的文章合集页面。

模块构成：

- 顶部标题区
- 当前标签文章数
- 排序切换
- 其他标签的横向 Rail
- 文章列表
- 分页器

### 8.6 归档页 `/archive`

定位：

- 体现博客积累与时间线感。

模块构成：

- 页面标题 + 总文章数
- 年份分组
- 每年下的文章列表
- 每项展示标题 + 日期

设计要求：

- 采用时间归档感强的排版。
- 可以轻微使用虚线或时间线辅助，但不要做传统简历式粗线。

### 8.7 文章详情 `/blog/[...slug]`

定位：

- 这是最重要的阅读页面，优先级最高。

模块构成：

- 日期
- 标题
- 标签区
- 正文内容
- 桌面端悬浮目录
- 阅读进度条
- 上一篇 / 下一篇导航
- 版权说明
- 评论区

正文设计要求：

- 阅读宽度控制在舒适范围。
- 代码块、图片、引用块、表格都要有统一视觉规范。
- 标题层级要非常清楚。
- 评论区要与正文拉开明显层级，但仍属于同一阅读流。

文章详情必须补齐的状态：

- 代码块
- 图片
- 引用块
- 有目录 / 无目录
- 有评论 / 无评论
- 深色模式

### 8.8 关于页 `/about`

定位：

- 展示个人资料、社交信息、技术栈和长文介绍。

布局建议：

- 桌面端两栏。
- 左侧固定资料区：头像、姓名、年龄文案、社交入口、技术栈图标。
- 右侧为 Markdown 正文。

视觉要求：

- 比首页更沉稳、更像个人名片页。
- 技术栈图标要规整，便于后台同步维护。

### 8.9 友链页 `/friends`

定位：

- 一半是“朋友们的站点展示”，一半是“交换友链入口”。

页面模块：

- 页面标题区
- 友链卡片网格
- 下方 Tab 切换
- Tab 1：我的博客信息
- Tab 2：申请友链表单

友链卡片应展示：

- 头像 / favicon
- 站点名称
- 简介
- 链接跳转

申请表单应展示：

- 名称
- 链接
- QQ
- 头像
- 简介
- 提交按钮

### 8.10 项目页 `/projects`

当前项目页还是占位状态，因此设计只需做一版“建设中 / 敬请期待”页面。

模块构成：

- 图标或插画
- 大标题
- 一段简短说明
- 小型装饰性点阵或节奏元素

### 8.11 全局搜索浮层

定位：

- 快速查找文章、标签、分类。

建议形态：

- 居中 Command Dialog
- 顶部搜索输入
- 下方结果列表
- 键盘快捷提示

### 8.12 建议反馈弹窗 / 抽屉

定位：

- 让访客快速给站长发送反馈。

桌面端：

- 居中圆角弹窗

移动端：

- 底部抽屉

模块构成：

- 标题区
- QQ 输入
- 建议内容输入
- 提交按钮
- 提交成功状态
- 失败提示状态

## 10. 后台页面详细说明

### 10.1 登录页 `/admin/login`

产品特征：

- 当前登录只输入密码，不输入用户名。
- 生产环境存在隐藏入口，不应在设计中做显式后台地址宣传。

页面构成：

- 品牌 Logo
- 品牌名
- 欢迎标题
- 说明文案
- 当前后台入口提示
- 密码输入框
- 显示 / 隐藏密码按钮
- 提交按钮
- 返回前台入口
- 左侧可保留轻量插画 / 动效角色

视觉建议：

- 登录页可以更有品牌温度，但仍需保持安全感。
- 不要做复杂多步骤登录。

### 10.2 仪表盘 `/admin`

定位：

- 第一屏看清“内容产出、互动变化、系统状态、下一步操作”。

模块构成：

- 顶部欢迎型 Summary Banner
- 6 张指标卡片
- 7 天趋势图
- 快捷操作卡片
- 6 个月热力图
- 系统状态卡片组

指标卡片建议展示：

- 本周文章
- 本周评论
- 本周建议
- 本月文章
- 本月评论
- 本月建议

系统状态建议展示：

- 运行时长
- 内存占用
- 数据库连接状态
- 最近刷新时间
- 内容总量摘要

### 10.3 文章管理 `/admin/posts`

定位：

- 对 Markdown 文件进行“运营式管理”。

模块构成：

- 顶部统计卡
- 搜索框
- 分类筛选
- 状态筛选
- 排序切换
- 已保存视图
- 保存当前视图
- 重置视图
- 批量操作条
- 文章表格
- 移动端文章卡片
- 分页

表格字段建议：

- 勾选框
- 文章标题
- 摘要
- 分类
- 标签
- 更新时间
- 字数
- 状态
- 操作按钮

批量操作：

- 批量发布
- 批量转草稿
- 批量改分类
- 批量改标签
- 批量删除

定位：

- 创作者高频核心页面，优先级非常高。

布局建议：

- 顶部操作栏
- 主编辑区
- 右侧元数据栏
- 桌面端支持“仅编辑 / 分屏 / 仅预览”
- 移动端自动改为纵向堆叠

主要模块：

- 返回文章列表
- 保存按钮
- 删除按钮
- 视图模式切换
- 前台预览按钮
- 标题输入
- Markdown 文本区
- 实时预览区
- Slug
- 发布时间
- 分类
- 标签
- 摘要
- 草稿开关
- 自动保存状态提示

必须补齐的交互状态：

### 10.5 评论审核 `/admin/comments`

定位：

单条线程卡片应展示：

## 12. 必须补齐的状态稿

Figma 不能只画“默认态”，至少要补齐这些状态：

- Focus
- Active
- Disabled
- Loading
- Empty
- Error
- Success
- Destructive Confirm
- Dark Mode
- Mobile

前台重点状态：

- 搜索无结果
- 建议提交成功 / 失败
- 友链为空
- 标签为空
- 文章无评论
- 404

后台重点状态：

- 仪表盘数据刷新中
- 表格无结果
- 评论待审核

- 友链不可达
- 编辑器自动保存恢复
- 删除确认

## 13. 原型流程建议

建议至少做以下 Prototype：

### 13.1 前台流程

- 首页 -> 博客列表 -> 文章详情 -> 评论区
- 首页 -> 搜索浮层 -> 文章详情
- 首页 -> 建议反馈弹窗 -> 提交成功
- 友链页 -> 申请友链 -> 提交成功

### 13.2 后台流程

- 登录页 -> 仪表盘
- 仪表盘 -> 新建文章 -> 编辑器 -> 保存 -> 前台预览
- 评论审核 -> 回复弹窗 -> 回复成功
- 建议收件箱 -> 套用模板 -> 回复 -> 状态变更
- 友链管理 -> 新增友链 -> 抓取元信息 -> 保存
- 关于页编辑 -> 实时预览 -> 保存
- 站点设置 -> 修改前台展示文案 -> 保存

## 14. 明确不要设计过头的内容

为了避免设计偏离当前项目范围，请不要额外扩展这些不存在的模块：

- 不需要设计多角色权限系统。
- 不需要设计媒体库中心。
- 不需要设计订单、支付、用户中心。
- 不需要设计复杂数据分析大屏。
- 不需要设计完整作品集管理后台，当前项目页只是占位页。
- 不需要把文章系统设计成富文本块编辑器，当前核心仍是 Markdown 文件编辑。

## 16. 给设计师的一句话总结

请把 Coet 设计成一个“有个人气质的技术博客 + 高效克制的内容运营后台”：

- 前台像一位审美在线、表达清晰的开发者名片。
- 后台像一张干净、顺手、信息密度高的创作者工作台。
- 整体风格统一，但前台更有人味，后台更有效率。
