# Coet 项目 Stitch 超详细设计提示词

你现在是一个资深产品设计师和全栈设计系统设计师，请为一个名为 `Coet` 的项目输出完整的前台与后台页面设计方案。你需要一次性给出高完整度、高可落地性的界面设计，而不是概念稿。请严格按照以下要求执行。

---

## 一、项目背景

`Coet` 是一个“个人技术博客 + 后台运营系统”一体化项目。

它的核心不是商城，不是企业官网，也不是通用 CMS，而是一个围绕个人内容表达、文章沉淀、评论互动、建议反馈、友链交换、站点运营而构建的个人站点系统。

系统由两部分组成：

- 前台：给访客阅读文章、了解站长、查看友链、提交评论、提交建议。
- 后台：给站长管理文章、审核评论、处理建议、维护友链、编辑关于页、配置站点设置。

注意：文章和关于页正文来自 Markdown/MDX 文件，不是块编辑器 CMS；后台的“文章编辑器”本质上是 Markdown 创作工作台。

---

## 二、设计目标

请把这个产品设计成：

- 前台：像一位有审美、有表达力、有技术沉淀的开发者个人品牌站。
- 后台：像一套高效、克制、顺手、卡片化的内容运营工作台。

设计目标必须同时满足：

- 足够专业
- 足够完整
- 足够可开发
- 足够统一
- 不要像模板站
- 不要像电商后台
- 不要像企业 OA
- 不要像过度营销的 Landing Page

## 四、视觉系统要求

### 1. 色彩

请建立完整设计变量，优先浅色模式，并兼容深色模式。

建议主色体系：

- 主色 `Blue`: `#2563EB`
- 辅助色 `Sky`: `#0EA5E9`
- 强调色 `Cyan`: `#06B6D4`
- 成功 `Green`: `#10B981`
- 警告 `Amber`: `#F59E0B`
- 危险 `Red`: `#EF4444`

建议中性色体系：

- 页面背景：`#F8FAFC`
- 一级卡片：`#FFFFFF`
- 二级底色：`#F1F5F9`
- 边框：`#E2E8F0`
- 主文字：`#0F172A`
- 次文字：`#64748B`
- 弱文字：`#94A3B8`

深色模式建议：

- 背景：`#020617`
- 卡片：`#0F172A`
- 次级卡片：`#111827`
- 边框：`#1E293B`
- 主文字：`#F8FAFC`

### 2. 字体

为中文、英文标题、代码分别建立字体策略：

- 中文正文：`Noto Sans SC`
- 英文 / 大标题：`Outfit`
- 代码 / 数据：`JetBrains Mono`

### 3. 圆角

请统一整个系统的圆角体系：

- 输入框 / 小组件：12
- 常规卡片：16
- 重点卡片：24
- 大容器 / 大弹窗：28-32
- 胶囊按钮：999

### 4. 阴影

阴影必须柔和、通透、轻量，不能厚重。

### 5. 质感

允许使用以下质感，但必须克制：

- 浅渐变
- 轻微玻璃感
- 柔和光晕
- 极浅点阵 / 网格背景
- 局部高亮边框

### 6. 动效

请在设计描述中体现这些动效方向：

- 页面进入：轻微上移 + 淡入
- Hover：轻微抬升、边框提亮、阴影增强
- 弹窗 / 抽屉：平滑出现
- 标签 / Tab / 筛选切换：滑块或高亮底板移动
- 不允许大面积炫技动效

## 七、前台页面详细要求

下面的每个页面都要高保真设计，不要省略细节。

### 页面 1：首页 `/`

#### 页面定位

这是整个产品最重要的品牌首页，要第一眼让人感受到：

- 这是个人技术博客
- 这是开发者品牌站
- 这是内容导向产品

#### 布局结构

桌面端采用 2 段式布局：

- 第一屏：Hero 全屏区域
- 第二屏：首页内容区

#### Hero 必须包含

- 左侧主标题：你好，我是 XXX
- 右侧大头像
- 角色说明：例如“全栈开发者”
- 一句短标语
- 社交图标组
- 向下滚动提示按钮

#### Hero 交互要求

- 社交图标 hover 有轻微上浮和 tooltip
- 头像区域有轻微氛围光
- 向下按钮点击后平滑滚动到“最新文章”区域

#### 首页内容区

请设计成左右布局：

- 左侧：最新文章列表
- 右侧：分类列表 + 热门标签

#### 最新文章区要求

- 顶部标题：最新文章
- 右上角：查看全部文章
- 每页显示 5 篇文章
- 文章列表项包括：
  - 发布时间
  - 标题
  - 摘要
  - 主分类
  - 标签
- 底部有分页切换按钮：上一页 / 下一页

#### 分类区要求

- 显示前 8 个分类
- 每个分类带数量
- hover 有轻微位移

#### 热门标签区要求

- 标签胶囊展示
- 每个标签带数量角标
- hover 有颜色变化和轻微缩放

#### 响应式要求

- 移动端 Hero 改为纵向居中
- 头像在上，标题和文案在下
- 最新文章和分类标签区改为上下堆叠

### 页面 2：博客列表 `/blog`

#### 页面目标

这是文章总索引页，要清晰、高效、可筛选。

#### 页面结构

#### 列表项设计

- 上一页 / 下一页

## 十二、原型流程要求

请在设计中体现以下可点击流程：

### 前台流程

- 首页 -> 博客列表 -> 文章详情 -> 评论区
- 首页 -> 搜索弹窗 -> 文章详情
- 首页 -> 建议反馈弹窗 -> 提交成功
- 友链页 -> 申请友链 -> 提交成功

### 后台流程

- 登录页 -> 仪表盘
- 仪表盘 -> 新建文章 -> 编辑器 -> 保存 -> 前台预览
- 评论审核 -> 回复弹窗 -> 回复成功
- 建议收件箱 -> 套用模板 -> 回复 -> 状态切换
- 友链管理 -> 新增友链 -> 抓取元信息 -> 保存
- 关于页编辑 -> 实时预览 -> 保存
- 系统设置 -> 修改前台文案 -> 保存
